# 生物多样性曲线优化问题实验报告

## 1. 项目框架概览

本项目围绕 EarthBench / CONOPLib 生物多样性曲线优化问题展开。问题的核心是：给所有物种的首现事件 FAD 和末现事件 LAD 排序，使该排序与地层剖面中的化石观测数据尽可能一致。解是一个长度为 `dims` 的排列，元素为 `0..dims-1`，其中偶数表示某个 taxon 的 FAD，奇数表示对应 LAD。评价函数由 CONOPLib 动态库计算 mismatch penalty，代码中取负数作为 fitness，因此 `y_best` 越接近 0 越好。

项目主要结构如下：

- `configs/`：Hydra 配置，包含任务配置和算法配置。
- `scripts/`：运行脚本，例如 `scripts/run_earth_124.sh`。
- `benchmarks/`：问题定义与黑盒评估，`benchmarks/CONOPLib_*/call.py` 调用 `CONOPLib.so`。
- `algorithms/`：优化算法实现，包括 EA、MAP-Elites、BO、RL 相关算法等。
- `Environments/`：强化学习算子训练/推理使用的环境。
- `main.py`：主训练入口，负责创建任务、实例化算法、Ray 并行评估、记录日志和保存结果。

当前实验主要沿 `scripts/run_earth_124.sh -> main.py -> algorithms/_map_elites.py -> algorithms/_ea_operator.py -> benchmarks/CONOPLib_124/call.py` 这条路径进行。每轮中，算法先通过 `ask()` 生成候选解，再由 `main.py` 中的 Ray worker 调用 benchmark 评估，最后通过 `tell()` 把 fitness 回传给算法更新 archive。

评估过程中，`EarthBenchmark.__call__()` 会先对候选解执行 `repair()`，修复 FAD/LAD 和 coexistence 约束，然后调用 `CONOPLib.so` 计算 penalty。为了保证保存的最优解合法，我们已将主流程改为保存 benchmark 返回的修复后解。

## 2. 交叉与变异算子

项目中的排列算子主要位于 `algorithms/_ea_operator.py`。

### 2.1 Mutation 算子

`swap_mutation`：随机选择两个位置交换。扰动较小，适合局部微调，但探索能力有限。

`insert_mutation`：随机选择一个元素，将其插入到另一个位置。该操作适合事件排序问题，因为它直接改变某个事件在整体年代序列中的位置，同时较好保留其余事件的相对顺序。

`reversal_mutation`：随机选择一段区间并反转。扰动较强，适合跳出局部最优，但可能较大幅度破坏已有结构。

`shuffle_mutation`：随机选择若干元素并打乱。探索性较强，适合增加多样性，但也可能引入较多无效扰动。

`shift_mutation`：在局部邻域内移动或交换某个元素。扰动范围较小，偏向局部搜索。

### 2.2 Crossover 算子

`order_crossover`：保留父代 1 的一个片段，其余位置按父代 2 中的相对顺序补齐。该方法强调保留相对顺序，适合排列优化。

`pmx_crossover`：部分映射交叉，保留父代 1 的片段，并用映射关系处理重复冲突。该方法能同时保留部分位置结构和片段结构。

`cycle_crossover`：根据两个父代之间的位置-元素循环进行继承，交替使用父代片段。该方法能保留父代中的位置关系，但有时变化幅度偏保守。

在 `earth_124` 实验中，baseline 使用 `pmx_crossover + insert_mutation`。从问题性质看，`insert` 对事件排序较自然，因此它本身就是很强的基础变异算子。

## 3. Baseline：MAP-Elites + 锦标赛选择

原始 MAP-Elites 的父代选择是从 archive 中随机选择两个解，没有利用已有 fitness。我们首先将父代选择改为锦标赛选择：

1. 从 archive 中随机采样 `tournament_size` 个候选父代。
2. 选择其中 fitness 最高者作为父代。
3. 对第二个父代重复该过程，并尽量避免与第一个父代相同。

该改动只利用 archive 中已有的 `fitnesses`，不会增加任何 benchmark evaluation 次数。实验中的 baseline 使用：

- `selection_type = tournament`
- `tournament_size = 3`
- `mutation_type = insert`
- `crossover_type = pmx`
- `correlation_threshold = 0.68`
- `archive_size = 20`
- `pop_size = 20`
- `init_sampler_type = init_v3`

baseline 完整运行结果位于：

`results/earth_124/map_elites/baseline/`

最终结果：

- `total_evaluations = 39964`
- `y_best = -3678`

## 4. 改进一：adaptive_mix mutation

固定使用 `insert` 虽然稳定，但可能探索不足。因此我们实现了 `adaptive_mix`，在以下变异算子之间动态选择：

- `swap`
- `insert`
- `reversal`
- `shuffle`
- `shift`

选择概率由两部分组成：

1. 阶段性先验：前期偏探索，提高 `shuffle/reversal` 权重；中期以 `insert` 为主；后期偏精修，提高 `insert/swap` 权重。
2. 历史成功率：记录每种 mutation 被尝试的次数，以及其生成的子代是否成功进入或替换 archive。成功率更高的算子后续获得更高权重。

该方法不会增加 evaluation 次数。每个候选解仍然只评估一次，只是在 `tell()` 阶段根据 archive 更新结果统计 mutation 的成功情况。

只加入 `adaptive_mix` 的实验结果位于：

`results/earth_124/map_elites/+adaptive_mix/`

最终结果：

- `total_evaluations = 39964`
- `y_best = -3687`

该结果弱于固定 `insert` baseline。说明在 `earth_124` 上，`insert` 本身已经非常适合该问题；额外加入 `shuffle/reversal` 等探索算子会带来噪声。尽管 adaptive 策略能自动调整概率，但在本次 seed 下并未超过纯 `insert`。

## 5. 改进二：动态 correlation_threshold 与局部搜索

MAP-Elites 的 archive 更新依赖 Kendall correlation。固定 `correlation_threshold` 会同时影响多样性和质量替换。我们实现了动态阈值：

- archive 未完成 warm-up 时，使用原始 `correlation_threshold`，避免 archive bootstrap 被卡住。
- archive 填满到 warm-up size 后，阈值从 `0.9 * correlation_threshold` 逐步增长到 `correlation_threshold`。

最初版本中，动态阈值从 `0.5 * correlation_threshold` 开始，导致前期过严，archive 长时间停在 1 或 2，实际 evaluation 数明显偏少。修正后，archive 能在前几轮正常填满。

我们还加入了轻量局部搜索：

- 每隔 `local_search.freq` 个 epoch 触发。
- 从当前 batch 里选 `top_k` 个高分候选。
- 对每个候选生成少量 `insert/swap` 邻居。
- 评估邻居并回传给 archive。
- 额外 evaluation 被记录在 `local_search_evaluations` 中。

该组合实验结果位于：

`results/earth_124/map_elites/+dynamic_threshold+local_search/`

最终结果：

- `total_evaluations = 40124`
- `local_search_evaluations = 160`
- `y_best = -3686`

该方法没有超过 baseline，反而略差。原因可能是局部搜索会把候选更快推向相似局部结构，而动态阈值虽然改善了 archive 更新节奏，但没有提供足够强的跨 basin 探索能力。

## 6. 改进三：LNS destroy-repair

为了增强跳出局部平台的能力，我们实现了 Large Neighborhood Search 风格的 destroy-repair 算子。该方法不是只移动单个 event，而是一次重构一小组 taxon：

1. 从候选解中随机选择若干个 taxon。
2. 删除这些 taxon 的 FAD 和 LAD 两个 event。
3. 将删除的 FAD/LAD 按随机顺序重新插回序列，并保证 FAD 位于对应 LAD 之前。
4. 生成的新候选仍然交给 `main.py` 的 Ray worker 统一 repair 和 evaluation。

该方法不会在算子内部调用 benchmark，因此不会隐藏增加 evaluation 次数。我们将 LNS 作为低频强扰动算子，并支持停滞后提高触发概率。

实验结果位于：

`results/earth_124/map_elites/+LNS/`

最终结果：

- `total_evaluations = 39964`
- `local_search_evaluations = 0`
- `y_best = -3687`

LNS 在本次实验中没有超过 baseline。其原因可能是当前实现是随机重构版本，并没有在大邻域中进行多候选筛选；它增加了扰动幅度，但扰动方向没有 fitness 引导，因此容易破坏 `insert + pmx` 已经形成的有效局部结构。

## 7. 改进四：threshold accepting 与 diversity bonus

进一步地，我们没有修改 benchmark 的 repair 逻辑，而是在 archive 接收规则上放松前中期约束。这样可以仍然只评价合法 repaired 解，同时允许部分略差但结构不同的候选保留下来。

### 7.1 Threshold accepting

原始规则中，当候选解与 archive 中某个个体过于相似时，只有候选 fitness 不低于该个体时才允许替换。我们加入 threshold accepting：

```text
fitness >= reference_fitness - threshold
```

其中 `threshold` 从 `threshold_accepting_start = 100.0` 线性衰减到 `threshold_accepting_end = 0.0`。前期允许略差解进入 archive，以帮助搜索穿过 fitness valley；后期逐步恢复严格选择。

实现时，为避免覆盖高质量 elite，我们区分两种情况：

- 若候选 fitness 更高，则仍然替换最相似 archive 个体。
- 若候选只是被 threshold 接受，则作为新 archive 成员加入；若 archive 超出容量，则由 archive 原有逻辑删除最差个体。

### 7.2 Diversity bonus

同时，我们给结构差异更大的候选一个逐步衰减的接收加分：

```text
accept_score = fitness + diversity_bonus_weight * (1 - max_correlation)
```

其中 `max_correlation` 是候选与 archive 中最相似个体的 Kendall correlation。`diversity_bonus_weight = 50.0`，并随 epoch 线性衰减到 0。该机制鼓励前期保留更多结构差异，后期回到 fitness 主导。

组合实验结果位于：

`results/earth_124/map_elites/+threshold_accepting+diversity_reward/`

最终结果：

- `total_evaluations = 39964`
- `local_search_evaluations = 0`
- `y_best = -3678`

该组合最终与 baseline 持平，没有突破 plateau。但它在早期更快达到较好的区间：达到 `y_best >= -3690` 的 epoch 为 634，而 baseline 为 970。说明放松 archive 接收门槛确实能提高前期探索效率，但本次实验仍未找到超过 `-3678` 的更优 basin。

## 9. 当前结论与不足

当前最稳定的基础方法仍然是 `tournament selection + insert mutation + pmx crossover`。`insert` 对 CONOP 事件排序问题非常合适，因此简单加入更强扰动并不一定能提升最终 best。

`adaptive_mix` 和 LNS 都没有超过 baseline。共同原因是它们增强了探索，但没有足够的问题结构或 fitness 引导；在 `earth_124` 上，随机大扰动容易破坏已有顺序结构。

动态 `correlation_threshold` 和局部搜索没有带来最终提升。局部搜索可能加快局部收敛，但也可能让 archive 更早集中到相似区域，降低后续跳出平台的能力。

threshold accepting 与 diversity bonus 是目前较稳的改动。它没有降低最终结果，且在早期更快达到较优区间，但仍未突破 `-3678` 的 plateau。后续可以进一步调小 threshold 或缩短衰减周期，避免前期过多保留较差候选。

EDA precedence restart 曾作为候选方向实现过，但当前实验效果不理想，因此不作为最终方法采用。一个可能原因是 EDA 从 archive elite 中学习到的 precedence matrix 主要复现当前 basin 的共识顺序，难以提供真正跨 basin 的新结构。

此外，目前主要结论仍基于单个 seed。后续实验应至少运行多个 seed，比较均值、最好值、标准差和收敛曲线，避免将随机波动误判为算法提升。
